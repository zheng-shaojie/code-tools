🔧 JSON字符串规范化功能

🚨 问题背景

在实际使用中，Excel单元格中的JSON字符串可能存在各种格式问题：

  • 单引号问题：JavaScript对象使用单引号，但JSON标准要求双引号
  • 属性名无引号：JavaScript对象的属性名可以不加引号
  • 尾随逗号：对象或数组最后一个元素后的多余逗号
  • JavaScript特殊值：undefined、函数、正则表达式等JSON不支持的类型
  • 注释：JSON标准不支持注释，但数据中可能包含
  • 格式混乱：多余的空格、换行等格式问题

✅ 解决方案

实现智能JSON规范化功能，自动修复常见的格式问题：

  • 自动修复：检测并修复多种非标准JSON格式
  • 向后兼容：支持标准JSON的同时兼容JavaScript对象格式
  • 错误提示：无法修复时提供详细的错误信息
  • 无损处理：尽可能保留原始数据的语义

📋 测试用例展示

🔧 单引号转双引号
修复前（无法解析）
{'name': 'John', 'age': 30}
❌ JSON.parse() 会抛出语法错误
修复后（可正常解析）
{"name": "John", "age": 30}
✅ 成功解析为标准JSON
🔧 属性名添加引号
修复前（JavaScript对象格式）
{name: "Alice", userId: 123, isActive: true}
❌ 属性名缺少引号，JSON标准不支持
修复后（标准JSON格式）
{"name": "Alice", "userId": 123, "isActive": true}
✅ 属性名已添加双引号
🔧 移除尾随逗号
修复前（包含尾随逗号）
{"items": ["apple", "banana", "orange",], "count": 3,}
❌ 尾随逗号导致解析失败
修复后（移除尾随逗号）
{"items": ["apple", "banana", "orange"], "count": 3}
✅ 尾随逗号已清除
🔧 处理JavaScript特殊值
修复前（包含undefined和函数）
{"name": "Bob", "value": undefined, "handler": function() { return true; }}
❌ undefined和函数不是有效的JSON值
修复后（转换为JSON兼容值）
{"name": "Bob", "value": null, "handler": "[Function]"}
✅ 特殊值已转换为JSON兼容格式
🔧 移除注释
修复前（包含注释）
{ "name": "Carol", // 用户姓名 /* 用户ID */ "id": 456, "active": true }
❌ JSON不支持注释语法
修复后（移除注释）
{ "name": "Carol", "id": 456, "active": true }
✅ 注释已移除，格式已优化
🔧 复杂嵌套对象修复
修复前（多种问题混合）
{ name: 'Dave', profile: { age: undefined, skills: ['JS', 'Vue',], // 技能列表 regex: /test/g, }, isAdmin: true, }
❌ 多种格式问题：无引号属性名、单引号、undefined、正则、尾随逗号、注释
修复后（全部问题已修复）
{ "name": "Dave", "profile": { "age": null, "skills": ["JS", "Vue"], "regex": "[RegExp]" }, "isAdmin": true }
✅ 所有格式问题已修复，可正常解析

🔨 功能特性

🔄
智能转换
自动识别单引号、无引号属性名等JavaScript对象格式，转换为标准JSON
🧹
格式清理
移除尾随逗号、注释、多余空格等影响解析的格式问题
🛠️
特殊值处理
将undefined、函数、正则表达式等非JSON值转换为兼容格式
📊
数值规范化
处理NaN、Infinity等特殊数值，转换为null或合适的表示
🔍
详细错误提示
当无法修复时，提供原始错误和规范化后的错误信息
⚡
性能优化
先尝试直接解析，失败后再进行规范化，减少不必要的处理

💻 技术实现

在JsonProcessor中新增了两个核心方法：

normalizeJsonString(jsonString) 规范化JSON字符串，修复各种格式问题
safeParseJson(jsonString) 安全解析JSON，自动应用规范化处理

处理流程：

  1. 首先尝试直接使用JSON.parse()解析
  2. 如果失败，应用normalizeJsonString()进行规范化
  3. 对规范化后的字符串再次尝试解析
  4. 如果仍然失败，提供详细的错误信息

集成方式：

所有现有的JSON处理方法（isJsonData、extractJsonData、getJsonDataType等）都已更新为使用safeParseJson()， 确保整个系统都能受益于JSON规范化功能。

🎉 现在可以处理各种非标准JSON格式了！

用户再也不用担心Excel中的JSON数据格式问题，系统会自动修复并正确解析。
